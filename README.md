# 中国朝代跨度图

把《朝代跨度表.xlsx》做成的交互网页版：纵轴为时间（每行 10 年，公元前 2030 — 公元 2030），
横轴为地理七大区（东南亚 / 青藏高原 / 西域 / 中原 / 东北 / 外蒙古 / 中亚），
每个政权是一块随疆域伸缩的不规则色块，配色沿用原表。

## 使用

直接双击打开 `docs/index.html` 即可（数据内嵌于 `data.js`，无需服务器）。
或起本地服务：

```bash
cd docs && python3 -m http.server 8000
# 打开 http://localhost:8000
```

### 交互

| 操作 | 效果 |
|---|---|
| 悬停色块 | 高亮整个政权，tooltip 显示起止年/时长/区域 |
| 点击色块 | 右侧详情面板：简介、原表批注、同期政权（可点击跳转） |
| 顶栏朝代条 | 平滑滚动到对应时代 |
| 搜索框 | 按政权名定位并打开详情 |
| ＋/－ 或 Ctrl+滚轮 | 三档缩放（概览/默认/细看） |
| ◐ | 深浅色切换 |
| URL 深链接 | `#y=618`（定位年份）、`#q=唐`（按名称；同名时列出选项）、`#p=158`（按条目 ID）、`?theme=dark` |

手机端支持横向滚动与双指缩放，详情面板变为底部卡片。

原表将末端画到 2030 年；网页展示不预测未来——网格与色块在当前年份截止，
仍存续的政权结束年显示为「今」，时长显示「至今约 N 年」。
当查证年代与原表色块跨度相差超过一个十年格时，详情和悬浮提示会同时列出
「年代」与「原表色块覆盖」，避免把两套口径混为一谈。

## 数据管线

```
朝代跨度表.xlsx ──(tools/extract.py)──▶ docs/data.json + docs/data.js
                                        tools/report.md (审核报告)
```

重新生成（改了 Excel 或修正规则后）：

```bash
uv run --with openpyxl tools/extract.py
```

- **名称修正 / 政权归并规则**：`tools/extract.py` 顶部的 `RENAMES` / `RENAME_AT` /
  `EXTRA_SEEDS` / `COLOR_ALIAS`，全部改动会列入 `tools/report.md` 供审核。
- **政权简介**（人工撰写，273 条）：`tools/intros.py`，键为政权名，
  重名政权用 `名称@起始年` 消歧；重跑解析不会丢失。
- **精确起止年**（查证自维基百科，236 条）：`tools/years.py`。
  绘图保持原表 10 年颗粒度；浮层与详情面板优先显示精确年份，
  查证不到的一侧回落为「约 + 十年颗粒」。
- 原始 Excel 已定稿不再修改，`extract.py` 视为一次性工具，产物已入库。
- 解析算法：同色相邻格子归并为连通块；带名字的合并单元格作「种子」，
  多种子连通块按非对称最近距离切分（政权向后延续便宜、向前追溯昂贵，
  即「政权存续至继任者出现」）；同名同色的分离块合并为一个政权的多个时段。

## 目录

```
├── 朝代跨度表.xlsx     原始数据
├── DESIGN.md          设计文档
├── tools/
│   ├── extract.py     解析脚本
│   ├── intros.py      政权简介（人工维护）
│   └── report.md      解析审核报告（自动生成）
└── docs/
    ├── index.html
    ├── css/style.css
    ├── js/app.js      轮廓描迹 + 渲染 + 交互（无依赖原生 JS）
    ├── data.json      解析产物
    └── data.js        同上（file:// 场景用）
```
