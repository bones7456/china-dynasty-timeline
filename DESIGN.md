# 中国朝代跨度图 · 网页版设计

> 数据源：`朝代跨度表.xlsx`（Sheet4）。目标：保留 Excel 版「时间 × 地理」二维长卷的形态，
> 做成 PC 优先、手机可用的静态网页，并新增交互。

## 1. 对原 Excel 的解读（数据模型依据）

- **纵轴 = 时间**：每行 10 年，行 2 起自公元前 2030 年，至 2030 年，共 409 行。
- **横轴 = 地理**：7 大区共 43 细列：
  东南亚(B–D) · 青藏高原(E–H) · 西域(I–L) · 中原(M–AE，19 列) · 东北(AF–AI) · 外蒙古(AJ–AM) · 中亚(AN–AQ)。
- **政权 = 不规则同色区域**：一个带名字的合并单元格「核心区」+ 周边同色散落格。
  色块宽度随时间变化 = 疆域扩张收缩（如唐、清），这是原表的精髓，网页版必须原样保留。
- 全表约 300+ 政权色块，颜色可完整提取；含 2 条批注（后赵石勒、冉魏冉闵）。
- 部分政权名竖排拆在相邻多格（如「李茂」+「贞」、「窦建」(德)、「刘黑」(闼)），解析时需按颜色归并后拼回全名。

## 2. 工程形态（已确认：小型静态项目）

```
shi/
├── 朝代跨度表.xlsx        # 原始数据（保留）
├── DESIGN.md
├── tools/
│   └── extract.py         # xlsx → data.json 解析脚本（uv run --with openpyxl）
└── docs/
    ├── index.html
    ├── css/style.css
    ├── js/app.js
    └── data.json          # 解析产物 + 人工补充（简介、修正）
```

- 无构建步骤，`python -m http.server` 或任意静态托管即可跑。
- `extract.py` 可重复执行；人工内容（简介、名称修正）放在独立的 `overrides` 数据段，
  重跑解析不会覆盖人工成果。

### data.json 结构

```json
{
  "meta": { "yearStart": -2030, "yearStep": 10, "columns": 43 },
  "regions": [ { "name": "中原", "colStart": 11, "colEnd": 29 }, ... ],
  "polities": [
    {
      "id": "tang",
      "name": "唐",
      "color": "#F37A29",
      "start": -2030 起算的行列矩形集合,
      "rects": [ [colStart, colEnd, rowStart, rowEnd], ... ],
      "note": "批注（如有）",
      "intro": "一两句简介（人工撰写）"
    }
  ]
}
```

同色且相邻（含跨合并区）的格子归并为同一 polity；同名同色但时间上断开的
（如秦在春秋与战国末的两段）也归并为一个政权、多段 rects。

## 3. 页面布局（PC 优先）

竖向时间轴的「历史长卷」，整页向下滚动：

```
┌────────────────────────────────────────────────────────┐
│  标题 ｜ 搜索 ｜ 朝代快捷导航（夏 商 周 秦 汉 … 清）｜ 缩放  │ ← sticky 工具栏
├─────┬───────────────────────────────────────────┬─────┤
│     │ 东南亚 │ 青藏 │ 西域 │   中   原   │ 东北 │ 蒙古 │ 中亚 │     │ ← sticky 区域表头
│ 年  │              ┌──────────┐                        │  年 │
│ 份  │              │  政权色块  │  ← SVG path 圆角描边     │  份 │
│ 轴  │              │ （名字竖排）│                        │  轴 │
│  ↓  │              └──────────┘                        │  ↓  │
└─────┴───────────────────────────────────────────┴─────┘
        滚动时右下浮现「公元前 xxx 年」当前年代指示器
```

- **渲染**：单个大 SVG。每个政权的格子集合合并为一条轮廓 path（圆角、微描边），
  名字竖排置于核心区；对商周汉唐这类长条，名字用 `position: sticky` 式跟随，
  滚到哪一段都能看到当前政权名。
- **左右年份轴、顶部区域表头**：sticky 固定；年份轴每 50 年一小刻度、100 年一大刻度，
  重要断代（如 -221、220、589、907、1279、1644、1912）加标签。
- **性能**：409 行 × 43 列规模不大，纯 SVG 足够，无需 canvas/虚拟化。

## 4. 交互（相对 Excel 的增强）

| 交互 | 行为 |
|---|---|
| 悬停政权 | 整块高亮、其余淡化；tooltip：名称、起止年、存续时长、占据大区 |
| 点击政权 | 右侧详情面板：起止年、存续时长、疆域大区、同期并存政权（可点击跳转）、批注、**一两句简介** |
| 快捷导航 | 顶栏主线朝代条，点击平滑滚动至该朝代并短暂高亮 |
| 搜索 | 输入政权名（支持拼音首字母可后续加），定位 + 高亮 |
| 缩放 | 3 档行高（概览 / 默认 / 细看），按钮 + Ctrl 滚轮 |
| 年代指示 | 滚动时浮层显示视口中心对应年份 |
| 深浅色 | 跟随系统，可手动切换（古卷底色有对应暗色版） |

**移动端**：横向可滚动（区域表头随动）、双指缩放、导航折叠进抽屉、
详情面板改为底部上滑卡片、色块名字按缩放级别自动降字号/隐藏。

## 5. 视觉（已确认：古卷纸质感）

- 底色米黄纸张纹理（暗色模式为深褐），政权色块沿用 Excel 原配色，
  统一做轻微降饱和以协调纸底。
- 标题、朝代名用衬线字体（思源宋体/系统衬线回退）；年份数字用等宽。
- 色块圆角 + 半透明深色描边，悬停时描边加深、微投影。
